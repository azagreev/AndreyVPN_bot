# Версионирование AndreyVPN_bot

## Единственный источник версии

Все версии определяются в `bot/version.py`:

```python
__version__ = "1.1.0"      # версия приложения
__schema_version__ = 1     # версия схемы БД (= число файлов миграций)
```

**Нигде больше версию не дублировать.**

## Семантическое версионирование

Формат: `MAJOR.MINOR.PATCH`

| Тип | Когда увеличивать | Пример |
|-----|------------------|--------|
| `MAJOR` | Несовместимые изменения (смена протокола, удаление команд) | `1.x.x → 2.0.0` |
| `MINOR` | Новые функции, обратно совместимые | `1.0.x → 1.1.0` |
| `PATCH` | Багфиксы, мелкие правки | `1.0.0 → 1.0.1` |

## Чеклист при выпуске новой версии

- [ ] Обновить `__version__` в `bot/version.py`
- [ ] Если изменилась схема БД — обновить `__schema_version__` и добавить миграцию
- [ ] Добавить запись в `CHANGELOG.md` (секция `[Unreleased]` → `[X.Y.Z] - YYYY-MM-DD`)
- [ ] Сделать git tag: `git tag -a v1.1.0 -m "Release v1.1.0"`

## Формат CHANGELOG

```markdown
## [1.2.0] - 2026-04-01
### Added
- Новые функции

### Changed
- Изменения в существующих функциях

### Fixed
- Исправленные баги

### Removed
- Удалённые функции (только при MAJOR)
```

## Версия схемы БД

`__schema_version__` **всегда равен** наибольшему `MIGRATION_ID` среди файлов в `bot/db/migrations/`.

При добавлении миграции `m002_*.py` с `MIGRATION_ID = 2`:
```python
__schema_version__ = 2   # увеличить вручную
```

Тест `test_version.py` проверяет это соответствие автоматически.

## Docker-образы

При сборке версия передаётся как build arg:
```bash
docker compose build --build-arg BOT_VERSION=1.1.0
```

Образы тегируются:
- `:latest` — текущая версия
- `:previous` — предыдущая (для отката, создаётся скриптом update.sh)
