# AndreyVPN_bot

Telegram-бот для управления VPN на базе **AmneziaWG**. Предоставляет пользователям доступ к VPN-профилям и позволяет администратору управлять пользователями, заявками и сервером.

---

## Docker

Рекомендуемый способ запуска. Подробная инструкция: [docs/DOCKER.md](docs/DOCKER.md)

```bash
cp .env.example .env
# Заполнить .env (BOT_TOKEN, ADMIN_ID, ENCRYPTION_KEY, SERVER_PUB_KEY, SERVER_ENDPOINT, WG_CONTAINER_NAME)
docker compose build && docker compose up -d
```

## Пользовательский интерфейс

После одобрения пользователь получает ReplyKeyboard:

```
🔑 Мои профили  |  📈 Трафик
ℹ️ Статус       |  ❓ Помощь
```

- **🔑 Мои профили** — список всех VPN профилей с inline-кнопками: 📥 .conf, 📱 QR, 🗑️ Удалить (с подтверждением). Кнопка ➕ Запросить новый профиль отправляет уведомление администратору.
- **📈 Трафик** — статистика потребления за текущий месяц по каждому профилю (сброс 1-го числа).
- **ℹ️ Статус** — информация об аккаунте: имя, username, ID, дата регистрации, количество профилей.
- **❓ Помощь** — инструкция по подключению.

Команды: `/start`, `/menu`

## Интерфейс администратора

Администратор открывает панель командой `/admin` и получает ReplyKeyboard:

```
👥 Пользователи  |  ⏳ Заявки
📊 Статистика    |  🖥️ Сервер
```

- **⏳ Заявки** — список ожидающих регистрации (пагинация по 5). Кнопки ✅ Одобрить / ❌ Отклонить. При одобрении пользователь получает уведомление с клавиатурой пользователя. Уведомления приходят в личку при новых заявках с inline-кнопками для мгновенного ответа.
- **👥 Пользователи** — полный список (пагинация по 5, ✅/❌ по статусу). Клик открывает детали: имя, username, ID, дата, список профилей. Действия: 🔑 Выдать VPN, 🚫 Заблокировать / ✅ Разблокировать. При блокировке/разблокировке пользователь получает уведомление.
- **📊 Статистика** — всего пользователей, одобрено, ожидает, VPN профилей, новых за сегодня и за неделю.
- **🖥️ Сервер** — статус WireGuard интерфейса (🟢 Работает / 🔴 Остановлен), количество активных пиров.

Администратор может использовать `/menu` для доступа к пользовательскому меню (например, чтобы проверить свои профили).

## Схема регистрации

```
Новый пользователь → /start → Капча → Ожидание одобрения
                                              ↓
                              Админ получает уведомление (⏳ Заявки)
                                    ↙              ↘
                            Одобрить              Отклонить
                               ↓                      ↓
                      Пользователь получает      Пользователь получает
                      клавиатуру + доступ        уведомление об отказе
```

## Схема получения VPN профиля

```
Пользователь → 🔑 Мои профили → ➕ Запросить → Уведомление админу
                                                        ↓
                                          Админ: ✅ Выдать конфиг
                                                        ↓
                                   Генерация ключей + выбор IP + синхронизация WG
                                                        ↓
                                   Пользователь получает QR + .conf файл
```

## База данных

Используется SQLite (через aiosqlite).

| Таблица | Назначение |
|---------|-----------|
| `users` | Зарегистрированные пользователи (`telegram_id`, `username`, `full_name`, `is_admin`, `is_approved`) |
| `approvals` | Заявки на доступ (`user_id`, `status`, `admin_id`) |
| `vpn_profiles` | VPN профили (`user_id`, `name`, `private_key` зашифрован Fernet, `public_key`, `ipv4_address`) |
| `daily_stats` | Ежедневная статистика |
| `configs` | KV-конфиги |

## Логирование

Используется [loguru](https://github.com/Delgan/loguru). Настройка производится в `bot/core/logging.py`, инициализация — один раз при старте через `setup_logging()`.

### Уровни

| Уровень | Номер | Назначение |
|---------|:-----:|-----------|
| `DEBUG` | 10 | Детали настройки: аргументы wg-команд, состояния FSM, загрузка конфига |
| `INFO` | 20 | Штатные события: регистрация, одобрение, выдача/удаление VPN профиля |
| `WARNING` | 30 | Неожиданное, но не критичное: пользователь заблокировал бота, ошибка уведомления |
| `ERROR` | 40 | Операция провалилась: wg-команда вернула ненулевой код, профиль не создан |
| `CRITICAL` | 50 | Бот не может работать: отсутствует `.env`, невалидный ключ |
| `AUDIT` | 25 | Кастомный уровень безопасности: кто что сделал с чьим доступом |

> Приватные ключи WireGuard **никогда** не попадают в логи.

### Файлы логов

Директория задаётся параметром `LOG_PATH` (по умолчанию `logs/`).

| Файл | Уровень | Ротация | Хранение | Назначение |
|------|---------|---------|----------|-----------|
| `stderr` | `LOG_LEVEL` | — | — | Цветной вывод в консоль |
| `logs/bot.log` | `LOG_LEVEL` | 10 МБ | 30 дней | Полная история событий |
| `logs/errors.log` | `ERROR` | 10 МБ | 90 дней | Только ошибки, с backtrace |
| `logs/audit.log` | `AUDIT` | 10 МБ | 365 дней | Журнал безопасности |

Все файлы сжимаются в `.gz` при ротации.

### Переключение уровня

По умолчанию `LOG_LEVEL=INFO`. Для первичной настройки и диагностики измените в `.env`:

```env
LOG_LEVEL=DEBUG
```

После этого в консоли и `bot.log` станут видны все wg-команды (`[WG] Команда: awg set awg0 peer ...`) и переходы FSM. Для возврата к штатному режиму верните `LOG_LEVEL=INFO` и перезапустите бота.

### Журнал безопасности (audit.log)

Каждое действие, затрагивающее доступ пользователей, записывается отдельной строкой:

```
2025-01-15 10:23:45 | AUDIT | REGISTER | user_id=123456 | username=@andrey
2025-01-15 10:24:12 | AUDIT | APPROVED | user_id=123456 | by_admin=999
2025-01-15 10:25:01 | AUDIT | VPN_ISSUED | user_id=123456 | profile=VPN_123456_1 | ip=10.8.0.2 | by_admin=999
2025-01-15 11:00:00 | AUDIT | VPN_DELETED | user_id=123456 | profile=VPN_123456_1
2025-01-15 12:00:00 | AUDIT | BLOCKED | user_id=123456 | by_admin=999
2025-01-15 12:05:00 | AUDIT | UNBLOCKED | user_id=123456 | by_admin=999
```

### Контекстные префиксы в логах

Сообщения в `bot.log` и консоли содержат префикс для быстрой фильтрации:

| Префикс | Источник |
|---------|---------|
| `[STARTUP]` / `[SHUTDOWN]` | Запуск и остановка бота (`main.py`) |
| `[REGISTRATION]` | Капча и регистрация новых пользователей |
| `[ACCESS]` | Блокировка / разблокировка пользователей |
| `[VPN]` | Создание, выдача и удаление профилей |
| `[WG]` | Вызовы awg/wg CLI (только при `LOG_LEVEL=DEBUG`) |

## Безопасность

- Приватные ключи WireGuard хранятся зашифрованными (Fernet).
- Доступ к боту только после ручного одобрения администратором.
- Защита от ботов через математическую капчу при регистрации. Команда `/cancel` позволяет выйти из процесса регистрации.
- `AdminFilter` на всех admin-хендлерах.
- Проверка владельца профиля в хендлерах `.conf`, QR и удаления — предотвращает скачивание чужих конфигов перебором `profile_id`.
- Все пользовательские данные (`full_name`, `username`) экранируются через `html.escape()` перед вставкой в HTML-сообщения.
- Журнал безопасности `audit.log` фиксирует все действия с доступом пользователей (кто, что, когда, от имени какого администратора).
- Приватные ключи WireGuard **никогда** не попадают в логи ни на каком уровне.
- Атомарность операций с WireGuard: профиль создаётся в БД только после успешной синхронизации с сервером; профиль удаляется из БД только после успешного удаления peer с сервера.
- Секреты (`.env`) не копируются в Docker-образ — передаются через переменные окружения.
- Лимит профилей на пользователя (`MAX_PROFILES_PER_USER`) предотвращает бесконтрольное создание VPN-профилей одним пользователем.
- Запрос на VPN имеет TTL 24 часа: если администратор не ответил, пользователь может отправить новый запрос автоматически.
- Конфигурация WireGuard (`VPN_IP_RANGE`, `SERVER_PUB_KEY`, `SERVER_ENDPOINT`) валидируется при старте бота — некорректные параметры приводят к немедленной остановке с понятным сообщением об ошибке.

## Параметры .env

| Параметр | Обязательный | Описание |
|----------|:-----------:|---------|
| `BOT_TOKEN` | да | Токен от @BotFather |
| `ADMIN_ID` | да | Telegram ID администратора |
| `ENCRYPTION_KEY` | да | Fernet ключ для шифрования приватных ключей WG |
| `DB_PATH` | нет | Путь к SQLite БД (по умолч. `bot_data.db`) |
| `WG_INTERFACE` | нет | Имя интерфейса (по умолч. `awg0`) |
| `WG_PORT` | нет | Порт WireGuard (по умолч. `51820`) |
| `SERVER_PUB_KEY` | да | Публичный ключ сервера |
| `SERVER_ENDPOINT` | да | `IP:PORT` сервера |
| `VPN_IP_RANGE` | нет | CIDR пул адресов (по умолч. `10.8.0.0/24`) |
| `DNS_SERVERS` | нет | DNS сервера (по умолч. `1.1.1.1, 8.8.8.8`) |
| `LOG_LEVEL` | нет | Уровень логирования: `DEBUG` / `INFO` / `WARNING` / `ERROR` (по умолч. `INFO`) |
| `LOG_PATH` | нет | Директория для файлов логов (по умолч. `logs`) |
| `REDIS_URL` | нет | URL Redis для хранения FSM-состояний (напр. `redis://localhost:6379/0`). Если не задан — используется MemoryStorage (состояния теряются при рестарте) |
| `MAX_PROFILES_PER_USER` | нет | Максимальное число VPN профилей на пользователя (по умолч. `3`) |
| `JC`, `JMIN`, `JMAX`, `S1`, `S2`, `H1`-`H4` | нет | Параметры обфускации AmneziaWG |
